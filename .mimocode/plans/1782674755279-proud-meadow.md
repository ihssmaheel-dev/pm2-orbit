# Process Grouping / Tagging ‚Äî Implementation Plan

## Goal
Add a tagging system that lets users assign color-coded tags to processes, filter by tags, group views by tag, and run bulk actions on tagged groups.

## Architecture

**Tags are stored server-side in `~/.pm2-orbit/tags.json`**, separate from PM2's data. The server merges tags into `ProcessSnapshot` before broadcasting via WebSocket. This keeps PM2 clean while making tags persistent across restarts.

### Data Model

```json
// ~/.pm2-orbit/tags.json
{
  "tags": [
    { "id": "t1", "name": "production", "color": "#ef4444" },
    { "id": "t2", "name": "api", "color": "#3b82f6" }
  ],
  "assignments": {
    "api-server": ["t1", "t2"],
    "worker": ["t1"]
  }
}
```

ProcessSnapshot gains an optional `tags` field:
```typescript
tags?: { id: string; name: string; color: string }[];
```

## Implementation Steps

### Step 1: Backend ‚Äî Types and Persistence
**Files:** `src/types/index.ts`, `src/core/persistence/tags.ts`

1. Add to `src/types/index.ts`:
   ```typescript
   export interface Tag {
     id: string;
     name: string;
     color: string;
   }
   export interface TagData {
     tags: Tag[];
     assignments: Record<string, string[]>; // processName -> tagIds
   }
   ```
2. Add `tags?: Tag[]` to `ProcessSnapshot` interface
3. Create `src/core/persistence/tags.ts`:
   - `loadTags(): TagData` ‚Äî reads `~/.pm2-orbit/tags.json`, returns defaults if missing
   - `saveTags(data: TagData): void` ‚Äî writes to disk
   - `getTagsForProcess(name: string): Tag[]` ‚Äî resolves tag IDs to full objects
   - `assignTags(processName: string, tagIds: string[]): void`
   - Cache in memory like settings.ts does

### Step 2: Backend ‚Äî API Routes
**File:** `src/routes/tags.ts`

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/tags` | GET | List all tags |
| `/api/tags` | POST | Create tag `{ name, color }` |
| `/api/tags/:id` | PUT | Update tag `{ name?, color? }` |
| `/api/tags/:id` | DELETE | Delete tag and all its assignments |
| `/api/tags/assign` | POST | Assign tags to process `{ processName, tagIds }` |
| `/api/tags/assign/:processName` | DELETE | Remove all tags from a process |

### Step 3: Backend ‚Äî Merge Tags into Snapshots
**File:** `src/core/pm2/bridge.ts`

Add a `tagResolver` function that the bridge calls when building snapshots:
- In `list()` and in the event pipeline tick, after getting ProcessSnapshot, merge `tags` field by looking up `assignments[proc.name]`
- Tags are resolved to full `{ id, name, color }` objects

### Step 4: Frontend ‚Äî Types and Store
**Files:** `ui/src/types/pm2.ts`, `ui/src/store/tags.ts`

1. Add `Tag` interface and `tags?: Tag[]` to frontend `ProcessSnapshot`
2. Create `ui/src/store/tags.ts`:
   ```typescript
   interface TagsStore {
     tags: Tag[];
     fetchTags: () => Promise<void>;
     createTag: (name: string, color: string) => Promise<void>;
     updateTag: (id: string, updates: Partial<Tag>) => Promise<void>;
     deleteTag: (id: string) => Promise<void>;
     assignTags: (processName: string, tagIds: string[]) => Promise<void>;
   }
   ```

### Step 5: Frontend ‚Äî Tag UI Components

2. `TagManager` ‚Äî dialog with:

   - Click a tag on a process ‚Üí assign/unassign toggle

### Step 6: Frontend ‚Äî ProcessTable Integration
**Files:** `ui/src/components/processes/ProcessTable.tsx`, `ProcessRow.tsx`

1. **Toolbar changes:**
   - Replace dead `statusFilter` with `tagFilter` in `useUIStore`
   - Add tag filter chips below search bar (clickable, toggle on/off)
   - Clicking a tag chip filters table to only processes with that tag
   - "Manage Tags" button opens `TagManager` dialog

2. **Table column:**
   - Add "Tags" column after Name (or render tags inline next to name)
   - Each cell shows `TagBadge` components for that process's tags

3. **Row interaction:**
   - Right-click or dropdown on a row ‚Üí "Assign Tags" submenu
   - Shows all available tags with checkmarks for current assignments

4. **Filtered bulk actions:**
   - When tag filter is active, bulk actions (Restart/Stop/Start All) target only filtered processes, not ALL processes

### Step 7: Group-by-Tag View Mode
**File:** `ui/src/components/processes/ProcessTable.tsx`

Add a toggle in toolbar: "List" / "By Tag" view modes.
- **List mode:** Current flat table
- **By Tag mode:** Collapsible sections, one per tag. "Untagged" section at bottom. Each section shows processes with that tag. Clicking section header collapses/expands.
